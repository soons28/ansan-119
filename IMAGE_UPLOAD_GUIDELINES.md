## 📌 현재 동작하는 아키텍처 (클라이언트 직접 업로드)
*   **업로드 방식**: 클라이언트(브라우저)에서 Supabase Storage로 직접 이미지를 업로드합니다.
*   **서버사이드 API 라우트 사용 금지**: 이전에는 서버측 API 라우트(`/api/upload` 등)를 거쳐 업로드하려 했으나, 환경 변수 설정의 복잡성 및 페이로드 크기 문제 등으로 인해 오류가 발생했습니다. 이를 우회하여 클라이언트 직접 업로드 방식으로 해결했습니다.
*   **렌더링**: PNG, HEIC, WebP 등 다양한 포맷의 이미지가 모바일 및 PC 환경에서 모두 정상적으로 렌더링됩니다.
